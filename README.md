# VLM на открытых данных VK

Дообучение Vision-Language модели на открытых данных коллекции [`deepvk`](https://huggingface.co/deepvk) и оценка на русскоязычных бенчмарках **GQA-ru** и **MMBench-ru**.

## Цель и задачи

**Цель.** Дообучить открытую VLM на открытых данных VK и проверить её на бенчмарках `GQA-ru` и `MMBench-ru`, стремясь к как можно более высокой метрике.

**Задачи.**
1. Настроить окружение (Google Colab, GPU T4, 16 ГБ VRAM).
2. Изучить датасеты: структура, примеры, размеры.
3. Взять за базу `deepvk/llava-gemma-2b-lora`, проверить инференс и влезаемость в T4.
4. Zero-shot оценка базовой модели на `GQA-ru` / `MMBench-ru` (метрика «до»).
5. LoRA-дообучение на подвыборке `deepvk/LLaVA-Instruct-ru`.
6. Повторная оценка, сравнение метрик до/после.

**Ожидаемый результат.** Работающий пайплайн дообучения VLM и таблица метрик «до/после» на двух бенчмарках, с честным анализом изменения качества.

## Как использованы открытые данные VK

Все данные и модель — из открытой коллекции VK `deepvk` на HuggingFace:

| Тип | ID | Как использован |
|---|---|---|
| Модель (база) | `deepvk/llava-gemma-2b-lora` | стартовая VLM (Gemma-2b + CLIP), дообучалась LoRA |
| Данные для обучения | `deepvk/LLaVA-Instruct-ru` | подвыборка 2000 диалогов для LoRA-дообучения |
| Бенчмарк 1 | `deepvk/GQA-ru` | оценка (открытый короткий ответ), 1000 вопросов |
| Бенчмарк 2 | `deepvk/MMBench-ru` | оценка (выбор A/B/C/D), 1000 вопросов |

Особенность `LLaVA-Instruct-ru`: в поле `image` лежит **путь на COCO**, а не сама картинка, поэтому нужные изображения `train2017` докачивались по URL с `images.cocodataset.org` (только для подвыборки).

## Модель и метод дообучения

- **База:** `deepvk/llava-gemma-2b-lora` — VLM (~3B) на языковой модели Gemma-2b и vision-башне CLIP ViT-L/14 (336px), обученная в LLaVA-сетапе.
- **Дообучение:** LoRA в fp16 на языковой части.
  - `r=16`, `alpha=32`, `dropout=0.05`, модули `q/k/v/o/gate/up/down_proj`.
  - vision-башня и проектор заморожены; обучаемых параметров ~19.6M (0.69%).
  - оптимизатор `adamw_torch`, LR `2e-4`, 1 эпоха, batch 1, grad-accum 16, gradient checkpointing.
  - чекпоинты — на Google Drive (resume после обрыва сессии).
- **Почему LoRA в fp16, а не 4-bit/QLoRA:** на актуальном окружении Colab (torch 2.4.1+cu121) 4-bit через bitsandbytes не инициализируется, а квантизация vision-башни ломает распознавание изображений. Модель 3B в fp16 занимает ~5.6 ГБ и свободно помещается в T4, поэтому 4-bit не требуется.

Итоговый артефакт — LoRA-адаптер (`adapter_model.safetensors`, ~84 МБ), применяемый поверх базовой модели.

## Выводы

- Полный пайплайн VLM отработан: разведка данных → загрузка и проверка модели → zero-shot базлайн → LoRA-дообучение с чекпоинтами → повторная оценка.
- Базовая модель `llava-gemma-2b-lora` уже была обучена на `LLaVA-Instruct-ru`, поэтому при дообучении на той же выборке train-loss держится на плато (~1.0–1.15) — модели почти нечего доучивать. Это ожидаемый и честный результат: заметного прироста на бенчмарках при дообучении на «уже виденных» данных может не быть.
- Возможные направления для роста метрики (на будущее): дообучение на данных, приближённых к формату бенчмарков (короткие ответы для GQA, выбор буквы для MMBench), больший объём/иные гиперпараметры, либо более крупная база `deepvk/llava-saiga-8b`.

## Как запускать

Ноутбуки из `notebooks/` открываются в Google Colab (T4 GPU). Порядок: `01 → 02 → 03 → 04 → 05`.
В начале каждого — установка версий выше, затем перезапуск сессии, логин `notebook_login()` (нужен HF-токен с read-доступом). Ноутбуки `04`/`05` требуют подключения Google Drive (чекпоинты/адаптер).
